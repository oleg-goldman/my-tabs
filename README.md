# My Tabs ⚡

Личная стартовая страница в духе [hello-nav](https://github.com/hello-nav/hello-nav):
плитки с моими ссылками, мгновенный поиск с клавиатуры, поиск по истории браузера,
тёмная тема. Синхронизация между компьютерами — через git, приватность — через
шифрование: наружу публикуется только зашифрованный блоб.

| | |
|---|---|
| **Сайт** | https://oleg-goldman.github.io/my-tabs/ |
| **Репозиторий** | https://github.com/oleg-goldman/my-tabs |
| **Стек** | чистые HTML/CSS/JS, без фреймворков и сборки |
| **Хостинг** | GitHub Pages (branch `main`, root) |

---

## Архитектура

Две части, использующие один и тот же код интерфейса (`app.js`, `style.css`):

1. **Сайт на GitHub Pages** — работает в любом браузере на любом компьютере.
2. **Chrome-расширение** (`extension/`) — подменяет новую вкладку тем же
   интерфейсом и добавляет недоступное сайту: поиск по истории браузера
   (`chrome.history`). Данные расширение тянет с сайта, поэтому после
   `git push` ссылки обновляются везде сами — расширения трогать не нужно.

### Поток данных

```
links.local.json ──npm run encrypt──▶ links.enc.json ──git push──▶ GitHub Pages
 (только локально,   PBKDF2 + AES-GCM    (в репозитории)                 │
  в .gitignore)      + favicon'ы внутрь                    ┌─────────────┴─────────────┐
                                                           ▼                           ▼
                                                   сайт (любой браузер)     расширение (новая вкладка
                                                                             + секция History в поиске)
```

### Файлы

| Файл | Назначение |
|---|---|
| `index.html` | страница сайта |
| `app.js` | вся логика: расшифровка, рендер, поиск, клавиатура, история |
| `style.css` | стили (флэт-дизайн hello-nav: иконки без карточек, градиентные заголовки, капсула поиска) |
| `links.local.json` | **ваши ссылки открытым текстом — только на вашем компьютере** (в `.gitignore`) |
| `links.example.json` | пример формата |
| `links.enc.json` | зашифрованные ссылки — единственное, что публикуется |
| `.favicon-cache.json` | кэш скачанных favicon'ов (в `.gitignore`) |
| `scripts/encrypt.mjs` | шифрование + скачивание favicon'ов |
| `scripts/decrypt.mjs` | восстановление `links.local.json` из блоба |
| `extension/manifest.json` | MV3: `chrome_url_overrides.newtab`, permissions `history` + `favicon`, host `oleg-goldman.github.io` |
| `extension/newtab.html` | страница новой вкладки (аналог `index.html`) |
| `extension/config.js` | `window.NT_REMOTE` — адрес, откуда расширение берёт `links.enc.json` |
| `extension/app.js`, `extension/style.css` | **копии** корневых файлов, обновляются `npm run build:ext` |
| `extension/icons/` | иконка-молния 16/48/128 px |

## Криптография

- Ключ: **PBKDF2-SHA256, 310 000 итераций** из пароля; шифр: **AES-256-GCM**
  (WebCrypto — одинаковый код в браузере и в Node-скриптах).
- Формат `links.enc.json`: `{ v, iter, salt, iv, data }`, всё в base64.
- **Соль переиспользуется** между перешифрованиями: обновление ссылок с тем же
  паролем не инвалидирует закэшированные ключи — браузеры пароль заново не просят.
  Свежая соль: `npm run encrypt -- --new-salt` (используйте при смене пароля).
- После первого ввода пароля производный ключ сохраняется в `localStorage`
  (`nt_key`/`nt_salt`) — дальше страница открывается без ввода. Если соль блоба
  не совпадает с сохранённой или расшифровка падает, пароль спрашивается заново.
- Пароль передаётся скриптам через скрытый ввод, `NT_PASSWORD` или `--password=…`.

### Модель угроз

Посторонний, открывший сайт или репозиторий, видит: код страницы, форму пароля
и зашифрованный блоб. Имена/адреса ссылок недоступны без пароля. Favicon'ы
встроены в блоб при шифровании, поэтому опубликованная страница не делает
внешних запросов и не раскрывает, какими сайтами вы пользуетесь. Страница
отдаётся с `noindex`. Ключ в `localStorage` защищён на уровне вашего устройства.

## Favicon'ы

`npm run encrypt` добавляет каждой ссылке иконку (data-URI) по цепочке:

1. сервис Google `s2/favicons` (64 px);
2. `https://<домен>/favicon.ico`;
3. разбор HTML главной страницы: `<link rel="…icon…">`.

Результат кэшируется в `.favicon-cache.json` (null-результаты перепрашиваются).
Если иконку найти не удалось — на странице рисуется цветная монограмма.
Можно задать свою: поле `icon` (URL или data-URI) в `links.local.json`.

## Ежедневное использование

### Изменить ссылки

```bash
# правите links.local.json, затем:
npm run encrypt                                   # спросит пароль
git add links.enc.json && git commit -m "links" && git push
```

Через ~минуту обновится везде: сайт и все расширения. Пароль заново не спросит.

### Формат

```json
{
  "categories": [
    {
      "name": "Work",
      "links": [
        { "name": "GitHub", "url": "https://github.com", "keywords": ["git", "код"] }
      ]
    }
  ]
}
```

`keywords` — синонимы для поиска (можно на любом языке), `icon` — необязательная
своя иконка. Подписи и заголовки лучше держать латиницей: заголовочный шрифт
(Avenir Next Condensed Italic) кириллицу рисует системным фолбэком.

### Поиск

Ранжирование: имя начинается с запроса → имя содержит → keywords → URL →
название категории. В расширении под результатами появляется секция **History**
(до 8 страниц из истории; ваши ссылки из неё исключаются). Устаревшие ответы
истории отбрасываются по токену запроса.

### Горячие клавиши

| Клавиша | Действие |
|---|---|
| любая буква или `/` | фокус в поиск |
| `↑ ↓ ← →` | выбор результата (сквозь обе секции) |
| `Enter` | открыть выбранное |
| `⌘Enter` / `Ctrl+Enter` | открыть в новой вкладке |
| `Esc` | сбросить поиск |

### Смена пароля

```bash
npm run encrypt -- --new-salt        # ввести новый пароль
git add links.enc.json && git commit -m "rotate password" && git push
```

Каждый браузер один раз спросит новый пароль.

## Установка на новом компьютере

### Просто пользоваться

Открыть https://oleg-goldman.github.io/my-tabs/, один раз ввести пароль.
Можно назначить домашней страницей.

### Расширение «новая вкладка»

```bash
git clone https://github.com/oleg-goldman/my-tabs.git
```

1. Chrome → `chrome://extensions`
2. Включить **Режим разработчика** (справа вверху)
3. **Загрузить распакованное расширение** → папка `my-tabs/extension`
4. `Cmd+T` → Chrome спросит, оставить ли новую вкладку от расширения → «Оставить»
5. Один раз ввести пароль

### Редактировать ссылки на новом компьютере

```bash
npm run decrypt        # восстановит links.local.json по паролю
```

## Обновления: что когда нужно делать

| Что изменилось | Сайт | Расширение |
|---|---|---|
| Ссылки (`links.enc.json`) | сам через ~1 мин | само при открытии вкладки |
| Интерфейс (`app.js`, `style.css`) | сам через ~1 мин | `npm run build:ext` → push; на компьютерах: `git pull` + «Обновить» на `chrome://extensions` |
| `manifest.json` | — | `git pull` + «Обновить» на `chrome://extensions` |

## Разработка

```bash
npm run serve          # локальный просмотр: http://localhost:4173
npm run build:ext      # синхронизировать app.js/style.css в extension/
```

Дизайн: флэт в духе hello-nav — иконки 46 px без карточек-контейнеров, ховер
мягкой подложкой (à la shadcn), заголовки секций Avenir Next Condensed Heavy
Italic с вертикальным серым градиентом, капсула поиска, светлая/тёмная темы
(автоматически по системе + ручной переключатель, хранится в `localStorage`).

## Troubleshooting

- **Страница снова просит пароль после обновления ссылок** — блоб перешифрован
  с новой солью (`--new-salt` или удалён старый `links.enc.json`). Это штатно:
  ввести пароль один раз.
- **У ссылки нет иконки** — домен не отдаёт favicon ни одним из трёх способов;
  задайте `icon` вручную или оставьте монограмму. Кэш можно сбросить, удалив
  запись из `.favicon-cache.json`.
- **Расширение показывает старый интерфейс** — `git pull`, `npm run build:ext`
  не забыт ли, затем «Обновить» на `chrome://extensions`.
- **`links.enc.json not found`** — не запускали `npm run encrypt` или файл
  не задеплоился (Pages собирается ~минуту после push).
